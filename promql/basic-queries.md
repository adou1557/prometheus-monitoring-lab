# Basic PromQL Queries

This document contains a few basic PromQL queries commonly used to monitor system health and resource utilization.

---

## Target Availability

Check whether a monitored target is reachable.

### Query

```promql
up
```

### Values

| Value | Status |
|---------|---------|
| `1` | Reachable |
| `0` | Unreachable |

### Description

The `up` metric is automatically generated by Prometheus and indicates whether the latest scrape of a target was successful.

---

## Available Memory

Display the amount of memory currently available on a host.

### Query

```promql
node_memory_MemAvailable_bytes
```

### Description

Shows the amount of memory available for applications, in bytes.

### Example

Convert the result to gigabytes:

```promql
node_memory_MemAvailable_bytes / 1024 / 1024 / 1024
```

---

## Available Disk Space

Display the available space on a filesystem.

### Query

```promql
node_filesystem_avail_bytes
```

### Description

Shows the available filesystem space, in bytes.

### Example

Convert the result to gigabytes:

```promql
node_filesystem_avail_bytes / 1024 / 1024 / 1024
```

---

## CPU Usage Rate

Measure CPU activity over time.

### Query

```promql
rate(node_cpu_seconds_total[5m])
```

### Description

Calculates the per-second rate of CPU time spent in each mode during the last 5 minutes.

### Notes

- Higher values indicate more CPU activity.
- Results are returned per CPU core and CPU mode (`user`, `system`, `idle`, etc.).

---

## Useful Label Filtering

PromQL allows filtering metrics using labels.

### Example

```promql
up{job="node-exporter"}
```

### Description

Returns only targets belonging to the `node-exporter` job.

---

## Summary

| Metric | Purpose |
|----------|----------|
| `up` | Check target availability |
| `node_memory_MemAvailable_bytes` | Monitor available memory |
| `node_filesystem_avail_bytes` | Monitor available disk space |
| `rate(node_cpu_seconds_total[5m])` | Measure CPU activity over time |
